Groupon Customer Service Tool

SUMMARY

A Groupon customer service agent process thousands of customer inquiries, issues & requests every day. Pizza is a chrome app that helps agent speed up live chat/email responses, keep track of multiple threads of conversations, and triage issues correctly and efficiently. I was asked to help incrementally improve the usability of MVP version and envision a more scalable future model. I was engaged in this project for a few months.


DATE

2020


COMPANY

Groupon


Role

Senior Product Designer




The Context


Pizza, the customer service tool, has become an essential part of our agents’ day-to-day operation; it allows agents to send pre-programmed snippets such as greetings, anwers to common questions and refund policies. The snippets can be called via macros(shortcut codes) or selected in a step-by-step flow. Agents also frequently look up the customer’s order info and utilizes the decision tree to determine the solution to a problem. Agents often service 2 or 3 customers simultaneously on a timer, as efficency is tied to their performance.

As I carefully studied the current tool through agent shadowing sessions and previous user reseach findings, I combed through my initial thoughts and mapped out an agent user flow. This was just for me was to understand the nuts and bolts. For example, which parts are connected and how they are connected, which parts are stand-alone.

There are a lot of potential improvements I could make for Pizza, but it wouldn’t be ideal to completely disrupt the current design agents are used to. It could feel a little overwhelming at first with so many moving parts and so much potential. One way I help everyone organize our thoughts is to segment our problems. The goal was to help the CX team envision a scalable framework to move forward with. The recommended model was left rail tabs. If new features were added, a new tab can be set up. This way the screen real estate is freed up. Another idea was to allow some customization within each tab.

To solve the 2 main issues with the current UI, I went in 2 directions. The first direction is to tokenize the visual styles. To reduce visual clutter, I recommended a very minimal visual system. For example, it only has 1 font size, 2 font weights, 4 greys and 4 status color. From there, mixing and match to create buttons, inputs, container, from bottom to top.

The second direction is to establish organization patterns that are logical and efficient.

One of the biggest challenge in this project is to accurately represent the design & functionalities. When designing a tool, there are a lot of nuances. Even though the goal for this phase is only to improve the UI, it’s hard to perceptively separate visual with functionality. To help the team grasp the full look & feel, I devised a prototype that highlights key interactions. Such as searching for a snippet and send to the chat.
